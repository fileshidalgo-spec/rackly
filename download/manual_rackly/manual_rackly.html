Sección 1: Introducción

1.1 ¿Qué es Rackly?

Rackly es un sistema integral de gestión de almacenes diseñado para optimizar el control de inventario en entornos de almacenamiento que utilizan estructuras de racks y áreas de piso. La plataforma permite registrar, consultar y administrar las mercaderías almacenadas en posiciones específicas dentro de un almacén, brindando visibilidad completa sobre la ubicación, cantidad y fecha de vencimiento de cada producto.

El sistema se divide en dos módulos principales: Kardex Racks, que gestiona las posiciones verticales de almacenamiento organizadas por bloques, torres, pisos y posiciones; y Kardex Piso, que administra las áreas de almacenamiento a nivel de suelo organizadas por sectores, columnas, subcolumnas, posiciones y niveles. Ambos módulos soportan operaciones de ingreso, salida, traslado y devolución de mercadería.

1.2 Propósito del Manual

Este manual tiene como propósito proporcionar una guía detallada y completa para que los usuarios del sistema Rackly puedan utilizar todas las funcionalidades disponibles de manera eficiente. El documento está dirigido a operarios de almacén, supervisores y administradores del sistema que necesiten realizar operaciones de control de inventario.

El manual sigue el formato de la American Psychological Association en su séptima edición (APA 7), garantizando una presentación profesional y estandarizada de la información.

1.3 Requisitos del Sistema

Para utilizar Rackly de manera óptima, se recomienda cumplir con los siguientes requisitos técnicos:

📱 Dispositivo

Dispositivo móvil con pantalla de al menos 5 pulgadas (smartphone, tablet). Se recomienda el uso en dispositivos móviles por la interfaz optimizada.

🌐 Navegador

Navegador web moderno: Google Chrome, Mozilla Firefox, Safari o Microsoft Edge (versiones actualizadas).

📶 Conexión

Conexión a Internet estable (Wi-Fi o datos móviles 4G/5G). Mínimo 1 Mbps de velocidad de subida y bajada.

👤 Cuenta

Credenciales de acceso proporcionadas por el administrador del sistema o registro previo en la plataforma.

✅ Recomendación

Se recomienda utilizar Rackly en un navegador móvil (Chrome para Android o Safari para iOS) para una experiencia óptima. La interfaz está diseñada con un enfoque mobile-first que prioriza la usabilidad en pantallas táctiles.

Sección 2: Acceso al Sistema

2.1 Inicio de Sesión

El inicio de sesión es el primer paso para acceder a las funcionalidades de Rackly. A continuación se describe el proceso paso a paso utilizando la pantalla de login del sistema.

Figura 1. Pantalla de inicio de sesión de Rackly.

Leyenda de anotaciones — Figura 1
① Campo de correo electrónico: ingrese su dirección de correo registrada.
② Campo de contraseña: ingrese su contraseña de acceso.
③ Botón «Ver contraseña»: alternar la visibilidad de la contraseña ingresada.
④ Botón «Ingresar»: inicia la sesión con las credenciales proporcionadas.
⑤ Pestaña «Registrarse»: accede al formulario de creación de cuenta nueva.

Procedimiento de inicio de sesión

1 Abra el navegador web en su dispositivo y diríjase a la URL proporcionada por el administrador del sistema.
2 En el campo Correo electrónico ①, ingrese la dirección de correo con la que fue registrado en el sistema.
3 En el campo Contraseña ②, ingrese su contraseña. Si necesita verificar los caracteres ingresados, pulse el botón Ver contraseña ③ para mostrar la contraseña en texto plano.
4 Pulse el botón Ingresar ④. El sistema validará sus credenciales y lo redirigirá al panel principal de Rackly.
5 Si las credenciales son incorrectas, aparecerá un mensaje de error indicando que el correo o la contraseña no son válidos. Verifique los datos e intente nuevamente.
⚠️ Nota importante

Si olvida su contraseña, comuníquese con el administrador del sistema para restablecerla. Rackly no permite la recuperación automática de contraseñas por motivos de seguridad.

2.2 Registro de Nuevo Usuario

Si es la primera vez que utiliza el sistema, deberá crear una cuenta de usuario. El proceso de registro le permite obtener credenciales de acceso individuales.

Figura 2. Pantalla de registro de nuevo usuario en Rackly.

Leyenda de anotaciones — Figura 2
① Campo de nombre completo: ingrese su nombre y apellido.
② Campo de correo electrónico: ingrese una dirección de correo válida.
③ Campo de contraseña: cree una contraseña segura (mínimo 8 caracteres).
④ Confirmación de contraseña: repita la contraseña para validar.
⑤ Botón «Registrarse»: envía el formulario y crea la cuenta.

Procedimiento de registro

1 En la pantalla de inicio de sesión, pulse la pestaña Registrarse ⑤ para acceder al formulario de registro.
2 Complete el campo Nombre completo ① con su nombre y apellido. Este dato se utilizará para identificarlo dentro del sistema.
3 En el campo Correo electrónico ②, ingrese una dirección de correo válida que será su identificador único de usuario.
4 En el campo Contraseña ③, cree una contraseña que contenga al menos 8 caracteres. Se recomienda combinar letras mayúsculas, minúsculas, números y caracteres especiales.
5 En el campo Confirmar contraseña ④, repita exactamente la misma contraseña para verificar que no haya errores de tipeo.

ℹ️ Información

Si el correo electrónico ya se encuentra registrado en el sistema, se mostrará un mensaje de error. En ese caso, intente iniciar sesión directamente o solicite asistencia al administrador.

Sección 3: Kardex Racks

El módulo Kardex Racks es el componente central de Rackly para la gestión de mercadería almacenada en estructuras verticales de racks. Este módulo permite realizar operaciones de ingreso, salida, traslado, devolución, consulta de stock y visualización de la ocupación del almacén.

3.1 Ingreso de Mercadería

La función de ingreso de mercadería permite registrar la entrada de productos al almacén, asignándolos a una posición específica dentro de la estructura de racks. Cada rack se identifica mediante cuatro coordenadas: Bloque, Torre, Piso y Posición.

Figura 3. Formulario de ingreso de mercadería en la pestaña Kardex Racks.

Leyenda de anotaciones — Figura 3
① Selector de Turno: permite elegir entre «Día» y «Noche» para identificar el turno de trabajo.
② Selector de Bloque: selecciona el bloque dentro del almacén.
③ Selector de Torre: define la torre vertical dentro del bloque seleccionado.
④ Selector de Piso: indica el nivel de altura dentro de la torre.
⑤ Selector de Posición: determina la posición horizontal dentro del piso.
⑥ Campo de búsqueda de producto: permite buscar por código o descripción del producto.
⑦ Campo de Cantidad: ingrese la cantidad de unidades a registrar.
⑧ Selector de Fecha de Vencimiento y casilla «Sin vencimiento».
⑨ Botón «Registrar Ingreso»: ejecuta el registro del ingreso.

Procedimiento de ingreso de mercadería

1 Acceda al módulo Kardex Racks desde el menú principal del sistema.
2 Seleccione el Turno ① correspondiente a su jornada laboral: «Día» o «Noche». Esta información queda registrada en el movimiento.
3 Seleccione el Bloque ② donde desea almacenar la mercadería. Utilice el menú desplegable para ver las opciones disponibles.
4 Seleccione la Torre ③ dentro del bloque elegido. Cada bloque puede contener múltiples torres.
5 Seleccione el Piso ④ que indica la altura dentro de la torre. Los pisos se numeran generalmente de abajo hacia arriba.
6 Seleccione la Posición ⑤ dentro del piso. Esta coordenada determina el lugar exacto en el nivel seleccionado.
7 En el campo de Búsqueda de producto ⑥, ingrese el código o la descripción del producto. El sistema mostrará sugerencias automáticas mientras escribe. Seleccione el producto deseado de la lista.
8 En el campo Cantidad ⑦, ingrese el número de unidades que serán almacenadas en la posición seleccionada.
9 Configure la Fecha de vencimiento ⑧ si el producto lo requiere. Utilice el selector de fecha para elegir la fecha correspondiente. Si el producto no tiene fecha de vencimiento, marque la casilla Sin vencimiento.
10 Verifique que todos los campos estén correctamente completados y pulse el botón Registrar Ingreso ⑨. El sistema procesará el registro y mostrará un mensaje de confirmación.
📋 Validación del sistema

El sistema verificará automáticamente que la posición seleccionada tenga capacidad disponible y que los datos ingresados sean válidos antes de procesar el ingreso. En caso de error, se mostrará un mensaje descriptivo.

3.2 Salida de Mercadería

La función de salida de mercadería permite registrar la extracción de productos del almacén. Esta operación reduce el stock disponible en la posición especificada y queda registrada en el historial de movimientos.

Interfaz de la pestaña Salida

Al seleccionar la pestaña Salida dentro de Kardex Racks, se despliega un formulario que permite buscar productos y registrar su extracción. La interfaz incluye los siguientes elementos principales:

  • Campo de búsqueda por código: Permite ingresar el código del producto para localizarlo rápidamente en el sistema. A medida que se escribe, se muestran sugerencias de coincidencia.
  • Selector de ubicación: Una vez seleccionado el producto, se muestran todas las posiciones donde existe stock del mismo, indicando la cantidad disponible en cada ubicación.
  • Campo de cantidad: Permite especificar la cantidad de unidades a extraer. El sistema validará que no se supere el stock disponible.
  • Información de stock: Cada posición listada muestra el stock actual, la fecha de vencimiento y el proveedor asociado al lote.

Salida masiva (múltiples posiciones)

El sistema soporta la salida masiva de mercadería, lo que permite extraer productos de múltiples posiciones simultáneamente. Para utilizar esta función:

  • Se muestran casillas de verificación (checkboxes) junto a cada posición con stock disponible.
  • Marque las casillas de las posiciones de las cuales desea extraer mercadería.
  • Especifique la cantidad a extraer de cada posición seleccionada.
  • Pulse el botón de confirmación para procesar todas las salidas de forma conjunta.

Procedimiento de salida de mercadería

1 Acceda al módulo Kardex Racks y seleccione la pestaña Salida.
2 Seleccione el Turno correspondiente a su jornada laboral actual.
3 En el campo de búsqueda, ingrese el código del producto que desea extraer. Seleccione el producto de las sugerencias mostradas.
4 Revise la lista de posiciones con stock del producto. Cada entrada muestra la ubicación (Bloque/Torre/Piso/Posición), la cantidad disponible, la fecha de vencimiento y el proveedor.
5 Para una salida individual, seleccione la posición deseada e ingrese la cantidad a extraer en el campo correspondiente.
6 Para una salida masiva, marque las casillas de verificación junto a las posiciones deseadas y especifique las cantidades para cada una.
7 Pulse el botón Registrar Salida para confirmar la operación. El sistema actualizará el stock en las posiciones afectadas y registrará el movimiento.
🚫 Stock insuficiente

Si intenta extraer una cantidad mayor al stock disponible en una posición, el sistema mostrará el mensaje «Stock insuficiente» y no permitirá completar la operación. Verifique las cantidades e intente nuevamente.

Sección 3: Kardex Racks (continuación)

3.3 Traslado

La función de traslado permite mover mercadería de una posición a otra dentro del almacén. Esta operación es útil para reorganizar el espacio de almacenamiento, consolidar productos o preparar pedidos. El sistema ofrece dos modalidades de traslado: Total (mover toda la cantidad) y Parcial (mover solo una parte del stock).

Vista de tarjetas móviles

En la pestaña Traslado, los productos y sus ubicaciones se presentan en formato de tarjetas adaptables a dispositivos móviles. Cada tarjeta muestra la siguiente información:

📌 Contenido de cada tarjeta

Coordenadas (B/T/P/Pos): Bloque, Torre, Piso y Posición de la ubicación actual.

Stock disponible: Cantidad de unidades actualmente almacenadas.

Fecha de vencimiento: Vencimiento del lote almacenado (si aplica).

Proveedor: Nombre del proveedor del lote.

Tipos de traslado

Modalidad Descripción Uso recomendado
Traslado Total Mueve toda la cantidad de mercadería de la posición de origen a la posición de destino. La posición de origen queda vacía. Cuando se necesita vaciar completamente una posición.
Traslado Parcial Mueve solo una parte de la mercadería. Se debe especificar la cantidad a trasladar. Ambas posiciones conservan stock. Cuando se necesita redistribuir stock sin vaciar la posición.

Procedimiento de traslado

1 Acceda al módulo Kardex Racks y seleccione la pestaña Traslado.
2 Seleccione el Turno de trabajo actual.
3 Busque el producto que desea trasladar ingresando su código o descripción en el campo de búsqueda.
4 Revise las tarjetas que muestran las posiciones con stock del producto. Identifique la posición de origen.
5 Seleccione la posición de destino utilizando los selectores de Bloque, Torre, Piso y Posición.
6 Elija la modalidad de traslado: Total o Parcial. Si selecciona Parcial, ingrese la cantidad a trasladar.
7 Pulse el botón Registrar Traslado para confirmar la operación. El sistema actualizará el stock en ambas posiciones.
✅ Confirmación visual

Después de un traslado exitoso, el sistema mostrará un resumen de la operación indicando la posición de origen, la posición de destino y las cantidades afectadas.

Sección 3: Kardex Racks (continuación)

3.4 Devolución

La función de devolución permite registrar la reentrada de mercadería que fue previamente extraída del almacén mediante una operación de salida. Esta operación es común cuando se devuelven productos no utilizados, rechazados por control de calidad o cuando se cancela un despacho.

Al registrar una devolución, el sistema incrementa el stock en la posición especificada y crea un registro en el historial de movimientos con el tipo «Devolución». Es importante indicar correctamente la posición donde se devuelve la mercadería para mantener la integridad del inventario.

Procedimiento de devolución

1 Acceda al módulo Kardex Racks y seleccione la pestaña Devolución.
2 Seleccione el Turno correspondiente a su jornada laboral.
3 Ingrese el código del producto que desea devolver utilizando el campo de búsqueda. Seleccione el producto de la lista de sugerencias.
4 Seleccione la posición de destino donde se reingresará la mercadería mediante los selectores de Bloque, Torre, Piso y Posición.
5 Ingrese la cantidad de unidades que se están devolviendo.
6 Si el producto tiene fecha de vencimiento, configure la fecha correspondiente. Si no aplica, marque la casilla Sin vencimiento.
7 Pulse el botón Registrar Devolución para confirmar. El sistema incrementará el stock y registrará la operación.
⚠️ Rastreo de devoluciones

Todas las devoluciones quedan registradas en el historial de movimientos con el identificador «Devolución». Esto permite mantener un registro completo y auditable de todas las operaciones realizadas en el sistema.

3.5 Consulta de Stock

La pestaña Stock dentro de Kardex Racks permite consultar el inventario actual de mercadería almacenada en las posiciones de racks. Esta funcionalidad es esencial para verificar la disponibilidad de productos, planificar reposiciones y realizar auditorías de inventario.

Opciones de búsqueda y filtrado

El módulo de consulta de stock ofrece las siguientes herramientas de búsqueda:

  • Búsqueda por código: Ingrese el código exacto del producto para localizarlo rápidamente.
  • Búsqueda por descripción: Ingrese parte de la descripción del producto para encontrar coincidencias parciales.
  • Filtros avanzados: Filtre por bloque, torre, piso o posición para ver el contenido de áreas específicas del almacén.

Vista de tarjetas con indicadores de vencimiento

Los resultados de la consulta se presentan en formato de tarjetas adaptadas para dispositivos móviles. Cada tarjeta incluye:

  • Información del producto: Código, descripción y ubicación completa (B/T/P/Pos).
  • Stock disponible: Cantidad de unidades en la posición.
  • Badges de vencimiento: Indicadores visuales de colores que alertan sobre la proximidad de la fecha de vencimiento del producto, siguiendo el sistema FEFO (First Expired, First Out).
    • Rojo: Producto vencido o próximo a vencer.
    • Amarillo: Producto con vencimiento intermedio.
    • Verde: Producto con vencimiento lejano o sin vencimiento.
ℹ️ Sistema FEFO

Rackly implementa la metodología FEFO (First Expired, First Out), que prioriza la salida de productos con fecha de vencimiento más cercana. Los badges de colores facilitan la identificación visual de productos que requieren atención prioritaria.

3.6 Ocupación (Mapa 3D)

La pestaña Ocupación proporciona una vista visual del estado de ocupación del almacén. Esta funcionalidad presenta un mapa tridimensional de las posiciones de racks, permitiendo identificar de un vistazo qué posiciones están ocupadas, vacías o contienen múltiples productos.

Cuadrícula visual de ocupación

El mapa de ocupación muestra una cuadrícula interactiva donde cada celda representa una posición dentro de la estructura de racks. Las celdas se codifican con colores para indicar su estado:

Posición vacía (disponible)
Posición ocupada (un producto)
Posición con múltiples productos

Interacción con el mapa

Al interactuar con las celdas del mapa de ocupación, se pueden realizar las siguientes acciones:

  • Toque simple: Muestra un resumen del contenido de la posición seleccionada (producto, cantidad, vencimiento).
  • Navegación por niveles: Utilice los controles para cambiar entre torres y pisos y visualizar diferentes secciones del almacén.
  • Vista general: Acceda a una vista consolidada que muestra el porcentaje total de ocupación del almacén.
✅ Beneficios del mapa de ocupación

El mapa de ocupación permite a los supervisores planificar la distribución del espacio de almacenamiento de manera eficiente, identificar rápidamente posiciones disponibles para nuevos ingresos y detectar áreas con alta densidad de productos que podrían requerir reorganización.

Operación Descripción Impacto en stock

Salida Extracción de mercadería del almacén. Disminuye (−)
Traslado Movimiento entre posiciones. Neutral (redistribuye)

Sección 4: Kardex Piso

El módulo Kardex Piso gestiona las áreas de almacenamiento a nivel de suelo del almacén. A diferencia de Kardex Racks, que utiliza una estructura vertical de bloques, torres, pisos y posiciones, Kardex Piso organiza las áreas de almacenamiento mediante una jerarquía diferente adaptada a las necesidades del piso.

4.1 Vista General

La estructura jerárquica de Kardex Piso se organiza de la siguiente manera:

🏗️ Jerarquía de Kardex Piso

Sector → Columna → Subcolumna → Posición → Nivel

El Sector es la división principal del área de piso. Cada sector contiene columnas, que a su vez se subdividen en subcolumnas. Las posiciones definen el espacio horizontal y los niveles indican la altura de apilamiento.

Kardex Piso funciona de manera independiente de Kardex Racks. Ambos módulos comparten la misma interfaz de usuario pero manejan estructuras de almacenamiento separadas, lo que permite una gestión diferenciada de mercadería en racks y en piso.

4.2 Sectores

La pestaña Sectores es el punto de entrada principal de Kardex Piso. Muestra una cuadrícula visual de todos los sectores del almacén, donde cada sector se representa como una tarjeta que contiene sus posiciones internas.

Cuadrícula de sectores con tarjetas de posición

Al acceder a un sector específico, se despliegan las tarjetas de posición que contienen la siguiente información:

  • Coordenadas completas: Columna, Subcolumna, Posición y Nivel.
  • Estado de ocupación: Indicador visual que muestra si la posición está vacía u ocupada.
  • Detalle de stock: Al tocar una tarjeta, se muestra el contenido detallado: producto, cantidad, fecha de vencimiento y proveedor.

Operaciones disponibles en Sectores

Desde la vista de sectores se pueden realizar las siguientes operaciones:

Operación Descripción
Ingreso Registrar entrada de mercadería en una posición de piso específica.
Salida Extraer mercadería de una posición de piso. Soporta salida masiva.
Traslado Mover mercadería entre posiciones de piso (total o parcial).
Devolución Reingresar mercadería previamente extraída de una posición de piso.

Salida masiva en Sectores

La funcionalidad de salida masiva permite seleccionar múltiples posiciones simultáneamente para extraer mercadería en una sola operación. Para activar esta función:

1 Acceda al sector deseado y visualice las tarjetas de posición.
2 Active el modo de selección múltiple pulsando el botón de Selección múltiple.
3 Marque las casillas de verificación (checkboxes) de las posiciones de las cuales desea extraer mercadería.
4 Especifique la cantidad a extraer de cada posición seleccionada.
5 Confirme la operación pulsando el botón de Registrar Salida Masiva.

Sección 4: Kardex Piso (continuación)

4.3 Stock Piso

La pestaña Stock Piso proporciona una vista global del inventario almacenado en todas las posiciones de piso. Permite consultar, buscar y filtrar la mercadería almacenada en las áreas de suelo del almacén.

Herramientas de búsqueda

  • Búsqueda por código: Ingrese el código del producto para localizarlo rápidamente en todas las posiciones de piso.
  • Búsqueda por descripción: Ingrese parte de la descripción del producto para encontrar coincidencias.
  • Filtros por sector: Filtre los resultados para mostrar solo las posiciones de un sector específico.

Badges de vencimiento (FEFO)

Al igual que en Kardex Racks, Stock Piso implementa indicadores visuales de vencimiento basados en la metodología FEFO (First Expired, First Out):

Color del badge Significado Acción recomendada
Rojo Producto vencido o con vencimiento inminente (menos de 7 días). Priorizar salida inmediata.
Amarillo Producto con vencimiento próximo (entre 7 y 30 días). Programar salida en el corto plazo.
Verde Producto con vencimiento lejano o sin fecha de vencimiento. Sin acción urgente requerida.

Comparación de stock con Big Magic

La pestaña Stock Piso incluye una funcionalidad de comparación de stock con Big Magic, que permite contrastar el inventario registrado en Rackly con los datos del sistema externo Big Magic. Esta función es útil para detectar discrepancias entre los dos sistemas y garantizar la consistencia del inventario.

⚠️ Discrepancias de inventario

Si la comparación con Big Magic muestra diferencias significativas, se recomienda realizar una verificación física del inventario y corregir las discrepancias mediante las operaciones de ingreso o salida según corresponda.

Sección 4: Kardex Piso (continuación)

4.4 Movimientos

La pestaña Movimientos registra y muestra el historial completo de todas las operaciones realizadas en Kardex Piso. Esta funcionalidad permite rastrear cada ingreso, salida, traslado y devolución con detalles como el usuario que realizó la operación, la fecha y hora, y las posiciones afectadas.

Filtros disponibles

El historial de movimientos cuenta con los siguientes filtros de búsqueda:

  • Filtro por tipo de operación: Seleccione el tipo de movimiento (Ingreso, Salida, Traslado, Devolución) para ver solo las operaciones de esa categoría.
  • Filtro por usuario: Seleccione un usuario específico para ver todas las operaciones realizadas por esa persona.
  • Filtro por rango de fechas: Defina un período de tiempo para visualizar solo los movimientos realizados dentro de ese intervalo.
  • Búsqueda general: Ingrese texto libre para buscar movimientos por código de producto, descripción o número de operación.

Contadores animados de resumen

En la parte superior de la pestaña Movimientos se muestran contadores animados que resumen la actividad del módulo Kardex Piso:

Contador Descripción Color
Total Número total de movimientos registrados en el período seleccionado. Gris oscuro
Ingresos Cantidad de operaciones de ingreso realizadas. Verde

Sección 5: Actualización en Tiempo Real

Rackly implementa un sistema de actualización en tiempo real que garantiza que todos los usuarios conectados vean la información más actualizada del inventario sin necesidad de recargar la página manualmente. Esta funcionalidad es fundamental en un entorno de almacén donde múltiples operarios realizan operaciones simultáneamente.

5.1 Mecanismo de funcionamiento

El sistema de actualización en tiempo real de Rackly utiliza dos tecnologías complementarias:

🔄 Polling (sondeo periódico)

El sistema realiza consultas automáticas al servidor cada 8 segundos para verificar si hay cambios en los datos del inventario. Este mecanismo asegura que, incluso si la conexión WebSocket se interrumpe temporalmente, los datos se mantendrán actualizados.

⚡ Supabase Realtime (WebSocket)

Adicionalmente, Rackly utiliza Supabase Realtime, un servicio basado en WebSockets que notifica instantáneamente a todos los clientes conectados cuando ocurre un cambio en la base de datos. Esto permite que las actualizaciones se reflejen de manera inmediata sin esperar al próximo ciclo de polling.

5.2 Sincronización multiusuario

Gracias al sistema de actualización en tiempo real, cuando un usuario registra un ingreso, salida, traslado o devolución, todos los demás usuarios conectados al sistema verán reflejado el cambio en sus pantallas de forma automática. Esto incluye:

  • Actualización de cantidades de stock en las posiciones afectadas.
  • Actualización de los contadores de movimientos en la pestaña de Movimientos.
  • Actualización del mapa de ocupación en la pestaña de Ocupación.
  • Aparición de nuevas entradas en el historial de movimientos.

5.3 No se requiere acción del usuario

El sistema de actualización en tiempo real funciona de manera completamente automática y transparente para el usuario. No es necesario realizar ninguna acción adicional para mantener los datos actualizados. Simplemente continúe trabajando normalmente y el sistema se encargará de sincronizar la información.

✅ Ventajas de la actualización en tiempo real

Prevención de conflictos: Reduce la posibilidad de que dos operarios intenten operar sobre la misma posición simultáneamente.

Información always-fresh: Garantiza que todos los usuarios trabajen con datos actualizados al momento de tomar decisiones.

Experiencia fluida: No requiere recargas manuales ni acciones del usuario para ver cambios.

Sección 6: Solución de Problemas

En esta sección se describen los problemas más comunes que pueden presentarse al utilizar Rackly, junto con sus soluciones correspondientes. Si el problema persiste después de intentar las soluciones sugeridas, comuníquese con el administrador del sistema.

6.1 Problemas frecuentes

Error: «Stock insuficiente» al registrar una salida
Causa: La cantidad solicitada supera el stock disponible en la posición seleccionada.
Solución: Verifique el stock actual en la posición consultando la pestaña Stock. Reduzca la cantidad de la salida o seleccione una posición diferente que tenga stock suficiente.
Error: «Correo o contraseña incorrectos» al iniciar sesión
Causa: Las credenciales ingresadas no coinciden con las registradas en el sistema.
Solución: Verifique que el correo electrónico esté escrito correctamente (incluyendo el dominio). Revise la contraseña asegurándose de no tener activada la tecla de mayúsculas. Utilice el botón «Ver contraseña» para confirmar los caracteres.
La pantalla no se actualiza con los últimos datos
Causa: Posible pérdida de conexión a Internet o interrupción del servicio WebSocket.
Solución: Verifique su conexión a Internet. Recargue la página del navegador (F5 o arrastrar hacia abajo en dispositivos móviles). El sistema restablecerá la conexión automáticamente.
El sistema responde con lentitud
Causa: Conexión a Internet débil o servidor bajo alta carga.
Solución: Verifique la velocidad de su conexión. Cierre otras aplicaciones que puedan estar consumiendo ancho de banda. Si el problema persiste, intente nuevamente en unos minutos.
No aparece un producto en la búsqueda
Causa: El producto puede no estar registrado en el catálogo o puede estar marcado como inactivo.
Solución: Verifique el código o descripción del producto. Si el producto es nuevo, solicite al administrador que lo registre en el catálogo mediante la pestaña Configuración.

6.2 Problemas de conexión

Rackly requiere una conexión estable a Internet para funcionar correctamente. Si experimenta problemas de conexión:

1 Verifique que su dispositivo esté conectado a una red Wi-Fi o tenga datos móviles activos.
2 Pruebe abriendo otra página web para confirmar que su conexión a Internet funciona correctamente.
3 Si utiliza Wi-Fi, intente acercarse al router o cambiar a datos móviles.
4 Recargue la página de Rackly para restablecer la conexión con el servidor.

6.3 Optimización para dispositivos móviles

Para garantizar la mejor experiencia de uso en dispositivos móviles, se recomienda seguir estas prácticas:

  • Navegador actualizado: Mantenga su navegador actualizado a la última versión disponible para asegurar compatibilidad con todas las funcionalidades de Rackly.
  • Caché del navegador: Limpiar el caché del navegador periódicamente puede resolver problemas de visualización. En Chrome: Configuración → Privacidad → Borrar datos de navegación.
  • Modo de pantalla completa: Utilice el modo de pantalla completa del navegador para aprovechar al máximo el espacio disponible en dispositivos pequeños.
  • Evitar múltiples pestañas: Cierre otras pestañas del navegador que no esté utilizando para liberar memoria y mejorar el rendimiento.
  • Notificaciones del navegador: Permita las notificaciones del navegador para Rackly si desea recibir alertas sobre actualizaciones importantes.
